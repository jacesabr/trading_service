1 · What this is

A self-operating research lab that discovers, validates, paper-tracks, and (after a gated, human-armed budget) live-trades short-horizon signals across many markets. A daily Claude Code cloud agent runs the lab via a guarded CLI; NVIDIA NIM is the always-on runtime LLM for in-code decisions; Firecrawl + Tavily bring web research and prior-art checks into context. The public sees results and a verifiable track record; an admin password reveals how each edge was found and used.

Two hard floors (never removed). Validity floor — nothing claims an edge without passing the leak test + purged walk-forward; the harness enforces NO-LOOKAHEAD by construction. Money floor — no real capital moves until a human arms a capped budget; hard caps + kill switch bound exposure. Everything else is open to explore.

4 · Strategy lifecycle

research ──leak PASS & walk-forward robust (≥4/5 windows, both instruments)──▶ paper
paper ──≥400 resolved & z≥2 (gate)──▶ live_candidate
live_candidate ──[HUMAN arms capped budget: LIVE_BUDGET_ARMED=1]──▶ live (auto, within caps)
any ──decay / drift / underperformance──▶ retired   (soft; record kept; never auto hard-delete)
  • lab promote refuses research→paper without a passing leaktest + walk-forward on record.
  • lab tweak resets a strategy to research and clears its track record — new params have not earned the old credibility. Re-validation required before money.
  • lab retire --reason is mandatory-reason and soft. Hard deletion is never automatic.

5 · The R&D harness (how NO-LOOKAHEAD is guaranteed)

Every prediction is made as_of bar t and scored on bar t+1's independent close. Two leak-free evaluation paths:

  • rule-vectorized — feature-threshold rules (e.g. meanrev/rules.json) over features(df, lag=False) in one vectorized pass. Fast, exact.
  • windowed-callable — universal fallback for arbitrary agent fns: a bounded trailing window ending at t, predict t+1. O(n).

Verified 2026-06-15: meanrev reproduces TEST 54–57% (z>2 both symbols) and is robust 5/5 walk-forward; the settled-negative wick_fade correctly fails (~47–49%, z<0). The engine separates the real edge from the dead control.

7 · The lab CLI

python lab.py list                         lifecycle + live state of every strategy
python lab.py status <name>                 full manifest + ledger + live stats
python lab.py new <name> --domain ... --kind binary --venue ... \
               --exec ... --adapter ... --symbols A,B --hypothesis "..."
python lab.py leaktest|backtest|walkforward|gridsearch <name>
python lab.py tweak <name> --set k=v --reason "..."   (resets track record)
python lab.py promote <name>                advance lifecycle iff its gate passes
python lab.py retire <name> --reason "..."  soft-retire (record kept)
python lab.py lesson --idea "..." --verdict rejected --evidence "..." --redo-bar "..."
python lab.py lessons                       the do-not-repeat memory (read first)

8 · Dashboard — public vs admin

  • Public / + /api/stats: results + identity + the verifiable track record (hit, equity, recent fills, entry prices). A whitelist (PUBLIC_KEYS) strips method/risk/params/rules/gate/provenance/ledger.
  • Admin /admin + /api/admin/stats: HTTP-Basic (ADMIN_USER/ADMIN_PASSWORD). Full manifest, exact params, how-found (hypothesis + research refs), experiment ledger, version history, lessons. Disabled (503) when no password is set — never open by default.
  • Docs /docs: this document, served live from docs/INFRA.html.
